Concrete visual & branding suggestions
Color palette (burner-themed)

Primary warm accents that map to “Burner Email” (dark mode first, then light tokens):

Background dark (near-black): #07120D (use for page background)

Surface / Card background (slightly lighter black/green tint): #0B1713

Primary accent (burner orange): #FF6B2E — use for primary CTAs, highlights, icons

Secondary accent (deep ember red): #FF3B30 — for destructive actions (delete) or error

Accent highlight (amber): #FFB86B — subtle glows / borders

Success (for toasts/ok): #2DD4BF (teal) — keep as reserved for success only

Muted text: #9AA8A0

Strong text: #E6F0EC

Why: orange/amber/red evoke fire and contrast beautifully on a very dark background. Using green as the main accent currently creates cognitive dissonance for the brand name.

Suggested CSS variables (example)

  --bg: #07120D;
  --surface: #0B1713;
  --text: #E6F0EC;
  --muted: #9AA8A0;
  --primary: #FF6B2E;
  --primary-weak: #FFB86B;

}

Typography / sizes (desktop)

Headline (H1): 34–40px, weight 700, letter-spacing -0.02em

H2: 26–30px, weight 600

Body: 15–16px, weight 400, line-height 1.6

Buttons: 14–15px, uppercase optional, padding 12px 20px

Microcopy (expiry timer): 12–13px, weight 500

Use a neutral geometric typeface for headings (Inter, Poppins, or Montserrat) and a more readable text face for body (Inter is fine across). Avoid too many font weights to keep build size small.

Inbox / Hero area — detailed suggestions
Problems seen

Large, heavy card with large padding which reduces scan speed.

Email string wraps into multiple lines in both desktop and mobile (bad for copy & paste).

CTA color is green and clashes with brand; there are too many similar actions (copy, refresh, change, delete) competing for attention.

Left and right gutters below the inbox are unused.

Fixes & layout

Condensed hero card: reduce vertical padding by ~30%, keep subtle rounded border, but remove heavy glow. Use a single-line email row with overflow: hidden; text-overflow: ellipsis; white-space: nowrap; and a copy icon to the right. Show full address on hover/tooltip and a “copy” success toast.

Primary CTA = “Copy” (prominent). Secondary actions small and ghost-styled. Example order: [Copy — primary] [Change] [Refresh] [Delete — danger]. Use --primary for Copy, and --danger for Delete.

Expiry timer: place under the email in muted text; make it clearer with small clock icon and readable color.

One-line inbox message listing: display email list items in a single-line row with subject, from, time — keep whitespace between left and right gutters for extra info panels (see below).

Accessibility & UX improvements

Contrast: Ensure text meets WCAG AA. For primary CTA on dark background, --primary on --surface should meet 4.5:1 for normal text; if not, use a subtle light outline or slightly lighter orange for text.

Keyboard: Tab order for copy, change, refresh, delete. Add aria-label attributes.

Tooltips: For icons (copy, refresh, QR), add accessible tooltips via aria-describedby or simple title.

Screen reader: Announce when an email is copied, deleted, or new mail arrives.

Performance & engineering suggestions

Reduce heavy shadows & big PNGs — use CSS rather than images for icons where possible (SVGs optimized).

Font loading: Use font-display: swap and subset fonts. Prefer system stack for body if you need speed.

Cache policies: Cache static assets with long TTL, invalidate on deploy.

Lazy-load non-critical images / below-the-fold content (testimonials, footer).

Bundle splitting: Ensure JS for inbox polling is small and tree-shaken; move analytics to async.

Optimize polling: If inbox polls frequently (5s in screenshot), use an adaptive approach: increase interval when idle, use server push / websockets if available.
